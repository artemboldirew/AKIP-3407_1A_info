# АКИП — одиночная гауссова волна через Python GUI

Этот проект содержит простую Windows-программу `akip_gaussian_wave_gui.py` для управления генератором АКИП через USB и выдачи одной гауссовой волны по нажатию кнопки.

Программа рассчитана на сценарий:

1. АКИП подключён к ноутбуку по USB.
2. Windows видит устройство через драйвер WCH CH372/CH375/CH376/CH378.
3. Python-программа загружает `CH375DLL64.dll`.
4. Через DLL отправляются текстовые SCPI-команды в АКИП.
5. После инициализации кнопка **«Дать одну GAUS-волну»** отправляет команду `*TRG`, и АКИП выдаёт один пакет GAUS.

## 2. Драйвер для обнаружения АКИП

Для того чтобы Windows обнаружила АКИП как USB-устройство через чип семейства CH37x, нужен драйвер WCH.

Официальная ссылка на драйвер:

https://www.wch-ic.com/downloads/CH372DRV_EXE.html

На странице WCH этот драйвер описан как драйвер для CH372/CH375/CH376/CH378, подходящий для Windows 11/10/8.1/8/7/Vista/XP и Windows Server, 32/64-bit.

После установки драйвера:

1. Подключите АКИП к ноутбуку.
2. Откройте **Диспетчер устройств Windows**.
3. Проверьте, что устройство появилось без жёлтого предупреждающего значка.
4. Если устройство не определяется, переустановите драйвер и переподключите USB-кабель.

---

## 3. Подключение АКИП к ноутбуку

Физическое подключение:

```text
АКИП USB-B  →  USB-A ноутбука
```

Обычно на стороне АКИП используется разъём USB-B, как у лабораторных приборов и принтеров. На стороне ноутбука используется USB-A.

Порядок подключения:

1. Включите АКИП.
2. Подключите кабель USB-B к АКИП.
3. Подключите USB-A к ноутбуку.
4. Дождитесь, пока Windows обнаружит устройство.
5. Запустите программу `akip_gaussian_wave_gui.py`.
6. Нажмите **«Инициализировать АКИП»**.

Важно: если АКИП не обнаруживается, проблема чаще всего связана не с Python-программой, а с драйвером, USB-кабелем, портом USB или состоянием самого прибора.

## 4. Инструкция по работе с `CH375DLL64.dll`

`CH375DLL64.dll` — это 64-битная DLL-библиотека, через которую Python-программа получает доступ к USB-устройству на базе CH375/CH37x.

### Куда положить DLL

Рекомендуемый способ:

```text
папка_проекта/
├── akip_gaussian_wave_gui.py
├── CH375DLL64.dll
└── README.md
```

То есть `CH375DLL64.dll` лежит рядом с `.py` файлом.

Альтернативный способ: выбрать путь к DLL через кнопку **«Выбрать...»** в интерфейсе программы.

### Разрядность

Важно, чтобы разрядность совпадала:

```text
64-bit Python  →  CH375DLL64.dll
```

Если используется 32-битный Python, 64-битная DLL не загрузится. В этом проекте предполагается использование 64-битного Python и `CH375DLL64.dll`.

### Какие функции DLL использует программа

Программа использует следующие функции DLL:

```text
CH375OpenDevice
CH375CloseDevice
CH375SetTimeout
CH375WriteData
CH375ReadData
```

Назначение:

- `CH375OpenDevice` — открыть USB-устройство.
- `CH375CloseDevice` — закрыть USB-устройство.
- `CH375SetTimeout` — задать таймауты обмена.
- `CH375WriteData` — отправить SCPI-команду в АКИП.
- `CH375ReadData` — прочитать ответ на SCPI-запрос, если команда была запросом с `?`.

Команды отправляются как ASCII-строки с окончанием строки:

```text
\r\n
```

Например команда:

```text
OUTP OFF
```

фактически отправляется как:

```text
OUTP OFF\r\n
```

## 5. Принцип взаимодействия с АКИП

Программа работает не напрямую с «кнопками» АКИП, а отправляет прибору команды дистанционного управления.

Общая схема такая:

```text
Python GUI
   ↓
ctypes.WinDLL
   ↓
CH375DLL64.dll
   ↓
USB-драйвер WCH
   ↓
USB-B / USB-A кабель
   ↓
АКИП
   ↓
SCPI-команды прибора
```

В интерфейсе пользователь задаёт:

- длительность GAUS-волны в миллисекундах;
- максимальный уровень `VOLT:HIGH`;
- режим расчёта уровня: напрямую в мВ или через `пик EEG × коэффициент делителя`;
- путь к DLL.

При инициализации программа настраивает АКИП:

1. выключает выход;
2. выключает Burst;
3. выбирает форму `GAUSsian`;
4. задаёт период и частоту;
5. задаёт нижний и верхний уровень напряжения;
6. включает Burst-режим по триггеру;
7. задаёт один цикл Burst;
8. выбирает внешний/программный запуск через `TRIG:SOUR EXT`;
9. включает выход;
10. оставляет прибор готовым к команде `*TRG`.

После этого нажатие кнопки **«Дать одну GAUS-волну»** отправляет только:

```text
*TRG
```

То есть форма, длительность и амплитуда уже должны быть заданы заранее на этапе инициализации.

## 6. SCPI-команды: общий принцип

SCPI-команды — это текстовые команды управления измерительными приборами.

Примеры:

```text
FUNC GAUSsian
VOLT:HIGH 200mV
OUTP ON
*TRG
```

Команды бывают двух типов:

### Команды установки параметров

Они задают состояние прибора:

```text
OUTP ON
BURS:STAT ON
VOLT:HIGH 200mV
```

### Команды-запросы

Они заканчиваются знаком `?` и ожидают ответ прибора:

```text
TRIG:SOUR?
BURS:STAT?
BURS:NCYC?
```

Программа использует запросы после инициализации, чтобы проверить, что критические параметры действительно применились.

---

## 7. Команды, в которых я уверен

Эти команды использовались в рабочей программе и сохранены в текущей версии GUI. Они являются основной проверенной последовательностью для выдачи одной GAUS-волны:

```text
OUTP OFF
BURS:STAT OFF
FUNC GAUSsian
FREQ <значение>Hz
PER <значение>ms
VOLT:LOW 0mV
VOLT:HIGH <значение>mV
BURS:MODE TRIG
BURS:NCYC 1
BURS:PHAS 0deg
BURS:STAT ON
TRIG:SOUR EXT
OUTP ON
*TRG
```

Также программа использует проверочные запросы:

```text
TRIG:SOUR?
BURS:STAT?
BURS:NCYC?
```

Ожидаемая логика:

- `OUTP OFF` — выключить физический выход перед настройкой.
- `BURS:STAT OFF` — выключить Burst перед перенастройкой.
- `FUNC GAUSsian` — выбрать гауссову форму.
- `FREQ ...Hz` — задать частоту, связанную с периодом.
- `PER ...ms` — задать период/длительность GAUS.
- `VOLT:LOW 0mV` — нижний уровень сигнала.
- `VOLT:HIGH ...mV` — верхний уровень сигнала.
- `BURS:MODE TRIG` — Burst запускается триггером.
- `BURS:NCYC 1` — один цикл Burst.
- `BURS:PHAS 0deg` — фаза Burst.
- `BURS:STAT ON` — включить Burst.
- `TRIG:SOUR EXT` — выбрать внешний/триггерный источник запуска, при этом в рабочей схеме команда `*TRG` запускает один пакет.
- `OUTP ON` — включить выход.
- `*TRG` — выдать одну волну.

---

## 8. Команды, в которых я не уверен

Эти команды **не следует считать проверенными для этой конкретной схемы**, пока они не протестированы отдельно на вашем АКИП и вашей прошивке.

```text
BURS:INT:PER <значение>s
TRIG:SOUR BUS
TRIG:SOUR IMM
TRIG:SOUR MAN
TRIG:SOUR INT
APPLy:GAUSsian ...
VOLT <значение>
VOLT:AMPL <значение>
VOLT:OFFS <значение>
FUNCtion GAUSsian
SOURce:FUNCtion GAUSsian
SOURce:VOLTage:HIGH <значение>
SOURce:VOLTage:LOW <значение>
SOURce:BURSt:STATe ON
SOURce:BURSt:NCYCles 1
SOURce:TRIGger:SOURce EXT
```

Почему они под вопросом:

1. У разных генераторов и разных прошивок могут отличаться полные и сокращённые формы SCPI-команд.
2. Некоторые SCPI-команды могут быть стандартными для одного семейства приборов, но не поддерживаться конкретной моделью АКИП.
3. В предыдущих тестах команда вида `BURS:INT:PER 500s` могла вызывать ошибку диапазона на части прошивок, поэтому она не используется в текущей рабочей последовательности.
4. Команды с префиксом `SOURce:` выглядят как стандартные SCPI-формы, но без отдельной проверки на вашем приборе их нельзя считать надёжными.

Для этой программы лучше использовать только короткие команды из раздела «Команды, в которых я уверен».

## 9. Важное замечание о делителе и полярности

Способ подключения делителя влияет на полярность сигнала, который в итоге попадает на вход EEG или на измерительный прибор.

Если поменять местами выводы делителя, землю, сигнальный провод или точку съёма сигнала, форма может визуально стать инвертированной:

```text
положительная GAUS-волна  →  отрицательная GAUS-волна
```

Это не обязательно означает, что АКИП генерирует неправильную форму. Часто причина именно в схеме подключения делителя, общей земле, направлении измерения или входе осциллографа/EEG.

Перед подключением к EEG обязательно:

1. проверить сигнал на осциллографе;
2. проверить реальную амплитуду после делителя;
3. проверить полярность;
4. убедиться, что нет опасного постоянного смещения;
5. убедиться, что вход EEG защищён и не перегружается.

## 10. Влияние предыдущего состояния АКИП

То, что было включено на АКИП до запуска программы, может влиять на работу дистанционного управления.

Например, до запуска программы на приборе могли быть включены:

- другой тип формы сигнала;
- другой Burst-режим;
- другой источник триггера;
- другой уровень напряжения;
- смещение;
- модуляция;
- режим Sweep;
- выход `OUTP ON`;
- внутренний период Burst;
- настройки, сохранённые с прошлого ручного запуска.

Поэтому программа при инициализации специально начинает с команд:

```text
OUTP OFF
BURS:STAT OFF
```

А затем заново задаёт критические параметры.

Тем не менее это не гарантирует полного сброса всех возможных режимов прибора. Если поведение странное, рекомендуется:

1. выключить выход АКИП;
2. вручную проверить, что не включены Sweep/Modulation/другие специальные режимы;
3. перезапустить АКИП;
4. заново запустить программу;
5. нажать **«Инициализировать АКИП»**;
6. проверить форму на осциллографе.

## 11. Безопасная процедура работы

Рекомендуемый порядок:

1. Установить драйвер WCH.
2. Положить `CH375DLL64.dll` рядом с программой.
3. Подключить АКИП по USB-B → USB-A.
4. Запустить программу.
5. Задать длительность GAUS.
6. Задать уровень `VOLT:HIGH`.
7. Нажать **«Инициализировать АКИП»**.
8. Проверить статус: АКИП готов.
9. Подключить выход АКИП к осциллографу через предполагаемый делитель.
10. Нажать **«Дать одну GAUS-волну»**.
11. Проверить амплитуду и полярность.
12. Только после проверки подключать систему к дальнейшей схеме.

## 13. Типичные ошибки

### Не найден `CH375DLL64.dll`

Решение:

- положить DLL рядом с `akip_gaussian_wave_gui.py`;
- или выбрать DLL через кнопку **«Выбрать...»**;
- проверить, что Python 64-bit.

### Не удалось открыть АКИП через CH375

Возможные причины:

- не установлен драйвер WCH;
- АКИП не подключён по USB;
- плохой USB-кабель;
- занят другой программой;
- не тот индекс устройства;
- прибор завис в предыдущем состоянии.

### Команда не передаётся

Возможные причины:

- DLL не соответствует системе;
- драйвер не установлен;
- прибор не отвечает;
- USB-соединение нестабильно;
- команда не поддерживается текущей прошивкой.

### Волна не появляется после `*TRG`

Проверьте:

- включён ли `OUTP ON`;
- включён ли `BURS:STAT ON`;
- стоит ли `BURS:NCYC 1`;
- выбран ли `TRIG:SOUR EXT`;
- не остался ли другой режим с прошлого ручного использования;
- правильно ли подключён осциллограф;
- не слишком ли маленькая амплитуда после делителя.

---

## 14. Важное предупреждение

Перед подключением к биологическим входам, EEG-усилителю или человеку необходимо использовать безопасную гальваническую развязку, корректный делитель, ограничение тока и независимую проверку осциллографом.

Эта программа только отправляет команды управления АКИП. Она не гарантирует электрическую безопасность подключённой схемы.
